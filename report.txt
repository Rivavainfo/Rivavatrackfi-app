# Google Play Sensitive Permissions Compliance Report

## Removed Permissions
*   `android.permission.RECEIVE_SMS`: Removed from `AndroidManifest.xml` as it is a sensitive permission restricted by Google Play and not essential for the core functionality.
*   `android.permission.READ_SMS`: Removed from `AndroidManifest.xml` for the same reason.
*   `android.permission.READ_CONTACTS`: Removed from dynamic permission requests in `SmsOptInScreen.kt`.

## Remaining Permissions
*   `android.permission.INTERNET`: Required for communicating with Firebase, external APIs (Alpha Vantage, Finnhub, Massive), and web scraping.
*   `android.permission.POST_NOTIFICATIONS`: Required on Android 13+ for showing local notifications related to app activity.

## Manifest Changes
*   Removed `RECEIVE_SMS` and `READ_SMS` `<uses-permission>` tags.
*   Removed the `<receiver android:name=".data.receiver.SmsReceiver">` block completely to ensure the app does not declare intent filters for `android.provider.Telephony.SMS_RECEIVED`, which could falsely flag the app as attempting to be a default SMS handler.

## Dependency Changes
*   Removed `org.tensorflow:tensorflow-lite` completely since the SMS text classifier features were removed, preventing dead code execution or 16 KB page size crashes.

## 16 KB Compatibility Status
*   **Compliant.** The app targets API 35, uses AGP 8.7.0, and sets `useLegacyPackaging = false`. The problematic TFLite dependency has been removed.

## Remaining Google Play Policy Risks
*   **None related to SMS/Call Logs.** The app no longer requests any sensitive SMS or Contact permissions. All traces of SMS parsing and tracking UI components and files have been removed.
